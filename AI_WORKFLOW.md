# Workflow IA

## Role de ce fichier

Ce fichier explique comment utiliser la documentation du projet avant toute modification importante.
Il sert de point de repere pour garder une bonne organisation et eviter de coder sans contexte clair.

## Ordre de lecture recommande

Avant chaque modification importante, il faut :

1. lire `AI_WORKFLOW.md` ;
2. identifier le type de modification a faire ;
3. lire ensuite le fichier Markdown concerne avant d'agir.

## Role de chaque fichier Markdown

### README.md

`README.md` est le fichier principal pour comprendre le projet.
Il donne la vue d'ensemble, la maniere de lancer le projet et l'etat actuel.

### PROJECT_CONTEXT.md

`PROJECT_CONTEXT.md` explique la vision generale du jeu.
Il clarifie le concept, la boucle principale et la direction du projet.

### GAMEPLAY.md

`GAMEPLAY.md` explique les regles du jeu.
Il definit les zones de tir, la logique d'arret ou de but, et les evolutions de gameplay prevues.

### ASSETS.md

`ASSETS.md` explique les images, sprites et animations necessaires.
Il aide a savoir quels elements visuels doivent etre prepares ou remplaces plus tard.

### TECHNICAL_PLAN.md

`TECHNICAL_PLAN.md` explique comment coder le projet.
Il precise les choix techniques, le role du HTML, du CSS, du JavaScript et la place de GSAP.

### TASKS.md

`TASKS.md` explique quoi faire et dans quel ordre.
Il sert de feuille de route simple pour avancer et verifier la progression.

## Regle de travail

Avant toute modification importante, il faut lire `AI_WORKFLOW.md`, puis le fichier Markdown le plus pertinent selon la tache a realiser.

Exemples :

- pour changer une regle du jeu, lire `GAMEPLAY.md` ;
- pour preparer des visuels, lire `ASSETS.md` ;
- pour organiser le code, lire `TECHNICAL_PLAN.md` ;
- pour savoir la prochaine etape, lire `TASKS.md`.
